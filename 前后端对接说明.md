# 前后端文件上传功能对接说明

## 📋 功能概述

本项目实现了完整的前后端文件上传功能，支持：
- ✅ 单文件上传
- ✅ 批量文件上传
- ✅ 拖拽上传
- ✅ 实时进度显示
- ✅ 上传状态反馈（成功/失败）

## 🚀 快速开始

### 1. 启动后端服务

```bash
cd KnowledgeBasePython

# 安装依赖
pip install fastapi uvicorn python-multipart aiofiles pydantic

# 或者使用项目配置
pip install -e .

# 启动服务
python main.py
```

启动成功后会显示：
```
🚀 启动 Knowledge Base 后端服务...
📖 API 文档：http://localhost:8000/docs
📝 ReDoc 文档：http://localhost:8000/redoc
INFO:     Uvicorn running on http://0.0.0.0:8000
```

### 2. 启动前端服务

```bash
cd KnowledgeBaseVue

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

### 3. 测试上传功能

打开浏览器访问前端地址（通常是 http://localhost:5173），然后：
1. 点击上传区域或拖拽文件
2. 查看上传进度和状态
3. 检查 `uploads/` 目录确认文件已保存

## 🔧 技术架构

### 后端 (FastAPI)
```
app/
├── routers/file_router.py      # API 路由
├── services/file_service.py    # 业务逻辑
├── schemas/file.py             # 数据模型
└── utils/logger.py             # 日志工具
```

**API 端点：**
- `POST /api/files/upload` - 单文件上传
- `POST /api/files/upload/batch` - 批量上传
- `GET /health` - 健康检查

### 前端 (Vue 3)
```
src/
├── components/FileUpload.vue   # 上传组件
├── utils/api.js                # API 配置
└── App.vue                     # 主应用
```

## 📝 API 接口说明

### 单文件上传

**请求：**
```http
POST /api/files/upload
Content-Type: multipart/form-data

file: <文件对象>
```

**响应：**
```json
{
  "filename": "example.pdf",
  "filepath": "uploads/example.pdf",
  "size": 1024,
  "message": "文件上传成功"
}
```

### 批量上传

**请求：**
```http
POST /api/files/upload/batch
Content-Type: multipart/form-data

files: [<文件对象 1>, <文件对象 2>, ...]
```

**响应：**
```json
[
  {
    "filename": "file1.pdf",
    "filepath": "uploads/file1.pdf",
    "size": 1024,
    "message": "文件上传成功"
  },
  {
    "filename": "file2.docx",
    "filepath": "uploads/file2.docx",
    "size": 2048,
    "message": "文件上传成功"
  }
]
```

## ⚙️ 配置说明

### 后端配置
- **上传目录**: `uploads/` (在项目根目录)
- **最大文件大小**: 默认无限制 (可在 Nginx 或其他层设置)
- **CORS**: 允许所有来源 (生产环境需限制)

### 前端配置
编辑 `src/utils/api.js`:
```javascript
export const API_CONFIG = {
  BASE_URL: 'http://localhost:8000',  // 修改为实际后端地址
  TIMEOUT: 30000,
}
```

## 🐛 常见问题

### 1. CORS 错误
**现象**: 前端报错 "Access-Control-Allow-Origin"
**解决**: 后端已配置 CORS 中间件，确保后端正常启动

### 2. 上传失败
**检查项**:
- [ ] 后端服务是否运行
- [ ] `uploads/` 目录是否存在且有写权限
- [ ] 文件是否为空

### 3. 前端无法连接后端
**解决方法**:
- 确认后端地址配置正确
- 检查防火墙设置
- 确认端口未被占用

## 📦 部署建议

### 生产环境配置

1. **后端**:
   - 使用 Gunicorn + Uvicorn workers
   - 配置 Nginx 反向代理
   - 限制上传文件大小
   - 设置域名白名单

2. **前端**:
   - 构建静态文件 `npm run build`
   - 使用 Nginx 或 CDN 部署
   - 配置正确的 API 地址
