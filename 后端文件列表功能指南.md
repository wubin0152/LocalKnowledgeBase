# 📋 后端文件列表功能 - 完整指南

## ✅ 已实现的功能

### 后端 API

#### GET `/api/files/list`
获取 uploads 目录中的所有文件列表

**请求：**
```http
GET /api/files/list
```

**响应：**
```json
[
  {
    "id": 1234567890,
    "name": "example.pdf",
    "size": 1048576,
    "type": "application/pdf",
    "filepath": "uploads/example.pdf",
    "uploadTime": "2026/03/22 22:30",
    "status": "success"
  },
  {
    "id": 1234567891,
    "name": "document.docx",
    "size": 2048000,
    "type": "application/vnd.openxmlformats-officedocument.wordprocessingml.document",
    "filepath": "uploads/document.docx",
    "uploadTime": "2026/03/22 22:31",
    "status": "success"
  }
]
```

**字段说明：**
- `id`: 文件唯一标识（基于路径的 hash 值）
- `name`: 文件名
- `size`: 文件大小（字节）
- `type`: MIME 类型
- `filepath`: 文件完整路径
- `uploadTime`: 上传时间（格式化字符串）
- `status`: 文件状态（始终为 "success"）

## 📊 完整的数据流

```
1. 用户打开前端页面
   └─> Vue 组件挂载 (onMounted)
   
2. App.vue 调用 loadFileList()
   ├─> 设置 loading = true
   ├─> 发送 GET 请求到 /api/files/list
   └─> 等待响应
   
3. FastAPI 接收请求
   ├─> file_router.get_file_list()
   ├─> 扫描 uploads/ 目录
   ├─> 读取每个文件的元数据
   └─> 返回文件信息列表
   
4. 前端接收响应
   ├─> 解析 JSON 数据
   ├─> 为每个文件添加 file: null 属性
   ├─> 赋值给 files.value
   └─> 设置 loading = false
   
5. FileList.vue 渲染
   ├─> watch 监听到 props.files 变化
   ├─> 输出调试日志
   └─> v-for 循环渲染文件列表
   
6. 用户看到
   └─> 文件列表展示在左侧面板
       - 加载动画 → 文件列表
       - 空状态 → 有文件
```

## 🐛 常见问题

### Q1: 文件列表为空？

**可能原因：**
1. uploads 目录确实为空
2. 后端服务未启动
3. CORS 配置问题

**解决方案：**
```bash
# 1. 检查 uploads 目录

ls uploads/

# 2. 如果有文件但 API 返回空，检查后端日志
# 查看终端中是否有错误信息

# 3. 上传一个测试文件
echo "test" > uploads/test.txt

# 4. 重新测试 API
python test_file_list.py
```

### Q2: 加载状态一直显示？

**可能原因：**
1. 后端 API 请求失败
2. 网络问题
3. 跨域问题

**解决方案：**
1. 打开 F12 查看 Console 错误
2. 检查 Network 标签中的请求状态
3. 确认后端地址配置正确

### Q3: 文件路径无法访问？

**问题描述：** 文件列表中显示了 filepath，但点击下载或预览时失败。

**解决方案：**
需要在后端添加文件下载接口：

```python
@router.get("/download/{filename}")
async def download_file(filename: str):
    """下载文件"""
    file_path = Path("uploads") / filename
    if not file_path.exists():
        raise HTTPException(status_code=404, detail="文件不存在")
    
    return FileResponse(
        path=file_path,
        filename=filename,
        media_type="application/octet-stream"
    )
```

## 🔧 高级功能

### 添加文件删除功能

在后端添加删除接口：

```python
@router.delete("/delete/{filename}", summary="删除文件")
async def delete_file(filename: str):
    """删除指定文件"""
    try:
        file_path = Path("uploads") / filename
        
        if not file_path.exists():
            raise HTTPException(status_code=404, detail="文件不存在")
        
        # 删除文件
        file_path.unlink()
        
        return {"message": "文件删除成功", "filename": filename}
        
    except Exception as e:
        raise HTTPException(
            status_code=500,
            detail=f"删除失败：{str(e)}"
        )
```

### 添加文件搜索功能

在前端已经实现的搜索基础上，可以添加后端搜索：

```python
@router.get("/search", summary="搜索文件")
async def search_files(q: str = ""):
    """搜索文件"""
    files = await get_file_list()
    
    if q:
        files = [
            f for f in files 
            if q.lower() in f["name"].lower()
        ]
    
    return files
```

### 添加分页功能

当文件很多时，可以添加分页：

```python
@router.get("/list", summary="获取文件列表")
async def get_file_list(page: int = 1, page_size: int = 20):
    """获取文件列表（支持分页）"""
    # ... 获取所有文件 ...
    
    # 分页
    start = (page - 1) * page_size
    end = start + page_size
    
    return {
        "files": files[start:end],
        "total": len(files),
        "page": page,
        "page_size": page_size
    }
```
