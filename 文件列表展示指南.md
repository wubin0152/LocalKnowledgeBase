# 📁 文件列表展示功能 - 完整使用指南

## ✅ 已实现的功能

### 1. 文件上传与展示流程
```
用户上传文件 
  → 调用后端 API 
  → 保存到服务器 
  → 返回文件信息 
  → 前端接收成功响应 
  → 添加到文件列表 
  → 实时展示在左侧面板
```

### 2. 文件列表功能特性

#### ✨ 视觉反馈
- **文件图标** - 根据文件类型自动显示对应图标
  - 📕 PDF 文档
  - 📘 Word 文档
  - 📊 Excel/PPT
  - 🖼️ 图片
  - 🎬 视频
  - 🎵 音频
  - 📝 文本文件
  - 💻 代码文件
  - 📦 压缩包

- **状态指示**
  - ✅ 上传成功：绿色左边框 + 渐变背景
  - ❌ 上传失败：红色左边框 + 渐变背景
  - ⏳ 上传中：进度条显示

- **详细信息**
  - 文件名（支持长文件名省略显示）
  - 文件大小（自动格式化为 B/KB/MB/GB）
  - 上传时间
  - 存储路径（后端返回的完整路径）

#### 🔍 搜索功能
- 实时搜索文件名
- 不区分大小写
- 即时过滤显示结果

#### 🎯 操作按钮
每个文件支持：
- 👁️ **预览** - 打开预览窗口查看内容
- ⬇️ **下载** - 下载文件到本地
- 🗑️ **删除** - 从列表中移除（带确认提示）

## 🐛 问题诊断

### 问题 1：文件列表为空

**检查点：**
1. 后端是否启动？访问 http://localhost:8000/health
2. 前端是否连接后端？查看 Console 是否有网络错误
3. 文件是否上传成功？查看 Console 日志

**解决方案：**
```javascript
// 在浏览器 Console 中运行以下命令检查状态

// 1. 检查后端连接
fetch('http://localhost:8000/health')
  .then(r => r.json())
  .then(d => console.log('后端状态:', d))
  .catch(e => console.error('后端异常:', e))

// 2. 检查当前文件数量
console.log('文件列表:', files)
```

### 问题 2：看不到上传的文件

**可能原因：**
1. 上传失败（查看 Console 错误）
2. 文件状态未正确更新
3. 组件间通信问题

**调试步骤：**
1. 打开 F12 开发者工具
2. 切换到 Console 标签
3. 上传一个文件
4. 查看所有日志输出

关键日志：
- `🚀 [FileUpload.vue] 开始上传文件...` - 表示上传开始
- `✅ [FileUpload.vue] 上传成功...` - 表示后端响应成功
- `📥 [App.vue] 收到上传的文件...` - 表示父组件收到数据
- `📋 [FileList.vue] 文件列表变化...` - 表示列表组件接收到数据

### 问题 3：CORS 错误

**错误信息：**
```
Access to fetch at 'http://localhost:8000' from origin 'http://localhost:5173' 
has been blocked by CORS policy
```

**解决方案：**
后端已在 `app/main.py` 中配置了 CORS：
```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],  # 允许所有来源
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

确保后端正常启动并重新加载页面。

## 📊 数据流详解

### 完整的上传流程

```
1. 用户操作
   └─> 点击上传区域 或 拖拽文件
   
2. FileUpload.vue 处理
   ├─> 创建文件对象 { id, name, size, type, file, progress: 0, status: 'uploading' }
   ├─> 添加到 uploadingFiles 数组（显示上传中的列表）
   └─> 调用 uploadToBackend()
   
3. 后端 API 调用
   ├─> POST http://localhost:8000/api/files/upload
   ├─> FormData: { file: <File> }
   └─> 等待响应
   
4. FastAPI 处理
   ├─> 接收文件
   ├─> 保存到 uploads/ 目录
   └─> 返回 { filename, filepath, size, message }
   
5. 前端接收响应
   ├─> 更新文件状态：status = 'success'
   ├─> 更新进度：progress = 100
   ├─> 保存文件路径：filepath = result.filepath
   └─> emit('filesUploaded', successFiles)
   
6. App.vue 接收
   ├─> handleFilesUploaded(successFiles)
   ├─> 添加到 files 数组
   └─> 触发响应式更新
   
7. FileList.vue 渲染
   ├─> watch 监听到 props.files 变化
   ├─> 输出调试日志
   └─> v-for 循环渲染文件列表
   
8. 用户看到
   └─> 文件出现在左侧列表
       - 文件图标
       - 文件名
       - 大小
       - 上传时间
       - 存储路径
       - 操作按钮
```

## 🔧 高级配置

### 修改上传目录
编辑 `KnowledgeBasePython/app/services/file_service.py`:
```python
def __init__(self, upload_dir: str = "uploads"):
    self.upload_dir = Path(upload_dir)  # 修改这里
```

### 修改 API 地址
编辑 `KnowledgeBaseVue/src/utils/api.js`:
```javascript
export const API_CONFIG = {
  BASE_URL: 'http://localhost:8000',  // 修改为实际地址
  TIMEOUT: 30000,
}
```

### 限制文件大小
在后端添加验证：
```python
MAX_FILE_SIZE = 10 * 1024 * 1024  # 10MB

async def upload_file(self, file: UploadFile) -> FileResponse:
    content = await file.read()
    if len(content) > MAX_FILE_SIZE:
        raise HTTPException(status_code=400, detail="文件大小超过限制")
    # ...
```
